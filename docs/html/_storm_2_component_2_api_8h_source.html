23/** This class define the interface for a Component. A Component is a data
24 driven 'block' that implements a function, algorithm, stuff, etc. Components
25 are designed to be run periodically. The application is responsible for

27 */
28class Api

30public:
31 /** This method completes any/all initialization for the Component. This
32 method MUST be called in the same thread context as the do() method.
33 The 'interval' argument is the periodic time interval between component
34 executions, e.g. a value of '50msec' means the component's do()
35 method will be executes its logic at 20Hz. This method should ONLY be
36 ONCE (without a call to stop()) and BEFORE any calls to do(); This
37 method return true when successfully; else false is returned.
38 */
39 virtual bool start( Cpl::System::ElapsedTime::Precision_T& intervalTime ) = 0;
40
41
42 /** This method is called to have a component perform its work. This
43 method MUST be called at least a frequently (if not faster) as the
44 desired/designed periodic rate of the component (see in the start()
45 call). Component are responsible for managing their own periodic
46 rate, i.e. this method should be called from the thread's top level
47 forever main() loop.
48
49 The 'enabled' argument provides a runtime/per-each-cycle option to
50 enabled or disable the Component's processing. When 'enabled' is
51 false this method does nothing.
52
53 The 'currentTick' argument represents the current elapsed time
54 since power-up of the application. The application should pass
55 the same 'currentTick' value to ALL components being executed in
56 the thread's main loop. This allows all component to synchronize
57 their execution with respect other components in main loop.
58
59 This method return true if the Component completed ALL of its
60 logic for the current processing cycle; else if an error occurred
61 then false is returned.
62 */
63 virtual bool doWork( bool enabled, Cpl::System::ElapsedTime::Precision_T& currentTick ) = 0;
