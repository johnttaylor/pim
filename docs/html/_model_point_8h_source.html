400  /** This method is used to attach a subscriber to a Model Point. Once
401  attached the Subscriber will receive a change notification (aka a
402  callback) every time the Model Point's data/state changes. Once, a
403  Subscriber is attached - it will stay attached to the application
404  calls detach().
405 
406  There is no limit to the number of Subscribers that can attach to
407  a Model Point.
408 
409  The attach() method can be called even if the Subscriber is already
410  attached. When this happens, the attach process is 'restarted', i.e.
411  the 'initialSeqNumber' is used for the Subscriber's sequence number.
412 
413  The change-detect mechanism uses a sequence number. Each Model Point
414  and each Subscriber has sequence number. When the Subscriber's sequence
415  number does not equals the Model Point's sequence number - the Subscriber
416  receives a change notification and the Subscriber's sequence number is
417  updated to match the Model Point's sequence number at the time of
418  change notification. When a Subscriber attaches to Model Point with
419  an the 'seqNumber' argument set to SEQUENCE_NUMBER_UNKNOWN, the
420  Subscriber will get an 'immediate' change notification.
421 
422  The callbacks for the Change Notifications are called as part of the
423  Data Model's Mailbox server. As part of the asynchronous processing (timers,
424  ITC, EventFlags, etc.) of the Data Model Mailbox server will also process
425  all pending Change Notification and invoke the call backs. What does
426  that all mean? The Change notifications are "local" to the Subscriber's
427  thread very similar to how the Cpl::Timers work. It also means that
428  no change notification callback will be called till the Mailbox server
429  loops back to the "top" of its forever loop.
430 
431  NOTE: The Change Notification mechanism does NOT guarantee that a
432  client will receive a notification for EVERY change to a Model
433  Point (i.e. 'fast edges' could/will be missed). What is guaranteed
434  is that once a Model Point's value has 'settle' all registered
435  subscribers will have received change notification for the Model
436  Point's current value.
