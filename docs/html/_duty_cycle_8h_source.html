23/** This static class provides methods to calculates on / off cycle times based
24 on the output of PI component.The PI output has linear mapping to duty cycle
25 as defined as :
26
27 dutyCycle = onTime
28 --------------
29 onTime + offTime
30
31 max - PI - Output = 100 % duty cycle.
32 max - PI - output / 2 = 50 % duty cycle.
33 zero - PI - output = 0 % duty cycle.
34 */
35class DutyCycle
36{
37public:
38 /** This method calculates the off cycle time in milliseconds based on the
39 provided 'process variable'. The mapping of the 'pvVar' is done
40 over of the range specified by the 'pvLowerBound' and 'pvUpperBound', e.g.
41 if the pvVar = 125 and pvLowerBound=100 and pvUpperBound=150, then the
42 'pvVar' input is normalized to 50% before the cycle time is calculated.
43
44 The calculated time will never be less than 'minOffTime'. However, the
45 specified 'minOffTime' may be lower based on the CPH setting to ensure
46 that the relationship between the Duty Cycle and the Process Variable
47 value does NOT have any horizontal regions (i.e. dead bands in the
48 mapping).
49
50 The 'cyclePerHour' argument is used to determine the 50% duty on/off
51 times. For example the 50% on/off times for 3CPH are 10min on/off.
52 */
53 static uint32_t calculateOffTime( float pvVar, uint32_t minOffTimeSec, Storm::Type::Cph cyclesPerHour, float pvLowerBound=0.0F, float pvUpperBound=100.0F );
54
55
56 /** This method calculates the on cycle time in milliseconds based on the
