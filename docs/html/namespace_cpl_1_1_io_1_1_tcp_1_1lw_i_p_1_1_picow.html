Detailed Description

The 'Picow' namespace implements the TCP interfaces for the Picow platform.

The dependencies for the implement are:

  • The Pico C/C++ SDK.

  • Built with the pico_cyw43_arch set to polling or thread_safe_background CYW43_LWIP=1 PICO_CYW43_ARCH_THREADSAFE_BACKGROUND=1 OR PICO_CYW43_ARCH_POLL=1

    Note: When building with PICO_CYW43_ARCH_POLL=1, all Wifi/TCP operation must be performed on the SAME core because of the SDK interrupt handling is not protected against multiple cores. Build with PICO_CYW43_ARCH_THREADSAFE_BACKGROUND=1 if you need to have Wifi/TCP operations execute on both cores.

  • Built with the lwIP config: NO_SYS=1

struct Cpl::Io::Tcp::lwIP::Picow::Socket_T

Defines the data/fields needed to manage a raw TCP connection.

The Application SHOULD treat this an opaque structure, i.e. never directly operate on it.

Class Members
void * connnectorClientPtr Pointer to the Connector client.

This field has no meaning if the Socket was created by a 'Listener'

struct tcp_pcb * lwipPcb Underlying lwIP Protocol Control block.

If null, there is no connection or an error occurred

struct pbuf * recvPbuf The PBUF pointer to the latest/current Receive data.

If null, then no incoming data. Note when there is socket error, this field is set to null

uint16_t rxOffset Starting offset when copying data from a PBUF into the client's buffer.
