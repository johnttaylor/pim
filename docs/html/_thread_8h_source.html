144 /** Internal Iterator that allows the Client to traverse the list

146
147 NOTES:
148 o There is difference between an "active" thread and a
149 Thread object instance. The underlying platform/OS
150 determines if the a thread is "active", i.e. the thread
151 is created and running. The Thread object gets created
152 BEFORE the thread is active and exists AFTER thread
153 terminates/end. Why the explanation, this method only
154 includes "active" threads. Threads that have terminated,
155 but that their associated Thread object instance has not

157 o This call is mutex protected. It will prevent new
158 threads from being created/deleted until it has

166 /** This method creates a thread according to the specified parameters. The
167 thread/task begins execution immediately upon its creation.
168
169 NOTE: Not all platform support all options, e.g. The C++11 multi
170 threading support does not directly support thread priorities.
171
172
173 @returns A pointer to the Thread created, OR 0 if the was an error.
174
175 @param runnable a runnable object which is started when the task is created
176 @param name a null terminated string that represent that thread's name
177 @param priority a priority ranging from highest priority (CPL_SYSTEM_THREAD_PRIORITY_HIGHEST)
178 to lowest priority (CPL_SYSTEM_THREAD_PRIORITY_LOWEST).
179 Priorities are defined in relative 'levels' to the max/normal/min
180 settings because there is no standardized priority settings
181 across Operating Systems. If the priority value exceeds the
182 upper/lower bounds the platform's priority range, the thread's
183 priority will be clamped to the upper/lower bound of the priority
184 range.
185 @param stackSize Requested size, in bytes, of the stack for the thread. A value of
186 of zero will size the thread's stack per the platform's default
187 stack size setting.
188 @param stackPtr Pointer to the stack memory to be used for the thread's stack. If
189 zero is passed, then the underlying platform/OS will allocate the
190 stack memory.
191 @param allowSimTicks With this argument is true the thread will be created to
192 use/support simulated time when 'Simulate ticks' are enabled.
193 A value false will force the thread to always be a 'real time'

195 simulated time.

206 /** This method is used to destroy a thread that was created by the Thread

208 Thread Factory, you MUST use the Factory's destroy method to delete it
209 to properly insure that the memory used to create the task is released
210 correctly.
211
212 NOTES:
213
214 o The application should only delete/destroy a thread AFTER the
215 thread's Runnable object.run() method has ended/terminated. If
216 the thread is destroy/delete before run() has completed, there is
217 NO GUARANTEE with respect to whether or not the thread has
218 released all acquired resources!
219 o NOT ALL Platforms support brute-force termination of threads,
220 e.g. in C++11 there is no method/interface to forcibly terminate
221 a thread. Once again -->it is a HIGHLY DISCOURAGED to design
