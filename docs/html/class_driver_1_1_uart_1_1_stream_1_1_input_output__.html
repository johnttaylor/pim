Detailed Description

This private concrete class implements the InputOutput stream interface using the UART stream driver.

A concrete sub class is required to provide the actual UART initialization component (i.e. the platform specific part of the stream driver.

The implementation ASSUMES that at most there is only ONE client attempt to use the InputOutput stream at any given time. It is okay to call the read(), write(), etc.from different threads - but the calls CANNOT be concurrent. It is the application's responsibility to provide additional thread-safety/concurrence protection. Note: The 'receiver' and the 'transmitter' methods MAY be called concurrently.
