39/** This concrete class implements a Input stream using Raspberry Pi UART
40 interface/hardware. This implement provide a more robust stream interface
41 than using 'stdio_uart' because it is interrupted driven and provides
42 software TX/RX FIFOs
43
44 Notes:
45 1. The UART Interrupt service handlers execute on the core that
46 executing start() method.
47 2. The implement does NOT support the blocking semantics of the Cpl::Io
48 streams. The design decision was for the implementation to be
49 compatible/useful on bare-metal systems, i.e. no threads required.
50
51 */
