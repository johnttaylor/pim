26/** This concrete class implements the core functionality for a Dictionary
27 and/or Hash Table. Key collisions are handled by a simple link list for
28 each 'hash bucket'. A default hashing algorithm is used, but the application
29 can supply a different/better algorithm. Clients should NOT USE THIS CLASS
30 DIRECTLY, but via the Dictionary class. The Map class is a wrapper
31 that makes the Hash Table type safe.
32
33 NOTE: This is no checking for duplicate keys. You can insert multiple
34 items with duplicates keys, but there is no guaranty on how
35 those items are found in searches or removed from the table.
36 */
